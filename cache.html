Project Overview

In this project, I implemented a caching simulation using the First-In-First-Out (FIFO) and Least Frequently Used (LFU) algorithms. The goal was to simulate the behavior of a cache by handling page requests and tracking cache hits and misses.

FIFO Algorithm Implementation


    def fifo():
        for p in requests:
            if p not in cache: 
                print('miss')
                cache.append(p)
            else: 
                print('hit')
            while len(cache) > 8:
                cache.pop(0)

To start, I created the `fifo()` function to implement the FIFO algorithm. Within this function, I looped through the page requests, checking if each page was already in the cache. If the page was not in the cache, it was considered a cache miss, and the page was added to the cache. Otherwise, it was a cache hit. If the cache exceeded its size limit (8 pages), the first page in the cache was removed using the FIFO replacement policy.

Next, I implemented the `lfu()` function to handle the LFU algorithm. Within this function, I used a frequency dictionary to keep track of the number of times each page was requested. Similar to the FIFO algorithm, I looped through the page requests, checking if each page was in the cache. If the page was already in the cache, it was a cache hit. If not, it was a cache miss. If the cache was not full, the page was added to the cache. However, if the cache was full, I used the LFU replacement policy to evict the least frequently used page from the cache. If multiple pages had the same least frequency of use, I chose the page that appeared first in the cache.

Main Loop and User Input

The `repeat()` function serves as the main loop of the program. It takes input from the user to request pages from memory. The user can enter page numbers (integers), and the requests are stored in the `requests` list. Entering '0' allows the user to stop entering requests and proceed with cache simulation.

Conclusion

The caching simulation project successfully implements the FIFO and LFU page replacement algorithms. It provides a user-friendly interface for testing the algorithms' behavior and allows users to observe cache hits and misses based on their page requests. The loop structure enables multiple cache simulations until the user decides to quit the program.
