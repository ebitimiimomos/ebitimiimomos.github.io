Project Overview

In this project, I designed and developed a web interface for a Sport Management Service. The interface allows users to interact with the service by sending HTTP requests and receiving responses. As part of this project, I implemented the following functionalities:

HTTP Method Selection

I created a dropdown menu where users can select an HTTP method (GET, POST, PUT, DELETE) for their request. The selection triggers corresponding actions based on the chosen method.

Resource URL Input

Users can input the resource URL in a text field. A datalist element provides them with pre-defined options for the resource URLs, facilitating easy selection.

Request Body Input

For methods other than GET, users can enter the request body in JSON format using a textarea field. The request body is sent with the corresponding HTTP request.

Sending HTTP Requests

Upon clicking the "Send Request" button, the web interface sends the HTTP request to the specified resource URL using the chosen method. It utilizes XMLHttpRequest to handle the asynchronous request and display the response.

Displaying HTTP Response

The response from the server is displayed on the web interface. It includes the HTTP status code, response body, and error messages (if any). The response body is formatted to make it more readable using JSON.stringify.

Clearing Response

A "Clear All Fields" button is available to clear the response and reload the page, enabling users to make another request.

Resource List and Examples

The interface includes a toggle button to view a list of available resources and their corresponding URLs. Examples of valid JSON bodies for different requests are also provided.

Conclusion

Throughout the project, I used HTML, CSS, and JavaScript to create the web interface and handle user interactions. The interface provides an efficient and user-friendly way for users to interact with the Sport Management Service, making HTTP requests and receiving responses in JSON format.
